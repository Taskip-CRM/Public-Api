# Project Management API

The Project Management API allows you to manage projects, tasks, milestones, and team collaboration in your TaskIP CRM system. You can create and track projects, assign tasks, monitor progress, and manage project workflows with comprehensive reporting and analytics.

## Endpoints Overview

### Project Management
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/public-v1/project` | Get all projects |
| `POST` | `/api/public-v1/project` | Create a new project |
| `GET` | `/api/public-v1/project/{id}` | Get a specific project |
| `PUT` | `/api/public-v1/project/{id}` | Update a project |
| `DELETE` | `/api/public-v1/project/{id}` | Delete a project |
| `POST` | `/api/public-v1/project/{id}/status-update` | Update project status |
| `POST` | `/api/public-v1/project/{id}/assign-team` | Assign team members to project |
| `GET` | `/api/public-v1/project/{id}/team` | Get project team members |
| `POST` | `/api/public-v1/project/{id}/duplicate` | Duplicate a project |
| `GET` | `/api/public-v1/project/{id}/analytics` | Get project analytics |

## Get All Projects

Retrieve a list of all projects in your system.

```http
GET /api/public-v1/project
```

### Request Headers
```http
X-Secret-Key: your-secret-key-here
Content-Type: application/json
```

### Query Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | No | Page number for pagination (default: 1) |
| `limit` | integer | No | Number of projects per page (default: 50) |
| `search` | string | No | Search term for project name or description |
| `sort` | string | No | Sort field (`name`, `status`, `start_date`, `end_date`, `created_at`) |
| `order` | string | No | Sort order (`asc` or `desc`) |
| `status` | string | No | Filter by status (`planning`, `active`, `on_hold`, `completed`, `cancelled`) |
| `client_id` | integer | No | Filter by client ID |
| `assigned_to` | integer | No | Filter by assigned team member |
| `priority` | string | No | Filter by priority (`low`, `medium`, `high`, `critical`) |
| `date_from` | string | No | Filter projects from start date (YYYY-MM-DD) |
| `date_to` | string | No | Filter projects to end date (YYYY-MM-DD) |

### Example Request

```bash
curl -X GET "https://public-api.taskip.net/api/public-v1/project?page=1&limit=20&status=active&priority=high" \
  -H "X-Secret-Key: your-secret-key-here" \
  -H "Content-Type: application/json"
```

### JavaScript Example
```javascript
const response = await fetch('https://public-api.taskip.net/api/public-v1/project?page=1&limit=20', {
  method: 'GET',
  headers: {
    'X-Secret-Key': 'your-secret-key-here',
    'Content-Type': 'application/json'
  }
});

const projects = await response.json();
```

## Create a New Project

Create a new project in your system.

```http
POST /api/public-v1/project
```

### Request Headers
```http
X-Secret-Key: your-secret-key-here
Content-Type: application/json
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Project name |
| `description` | string | Yes | Project description |
| `client_id` | integer | Yes | Client ID for the project |
| `project_manager_id` | integer | Yes | Project manager user ID |
| `start_date` | string | Yes | Project start date (YYYY-MM-DD) |
| `end_date` | string | Yes | Project end date (YYYY-MM-DD) |
| `budget` | decimal | No | Project budget |
| `priority` | string | No | Priority level (`low`, `medium`, `high`, `critical`) |
| `status` | string | No | Initial status (`planning`, `active`) |
| `team_member_ids` | array | No | Array of team member IDs |
| `tags` | array | No | Array of project tags |
| `custom_fields` | object | No | Custom field values |
| `template_id` | integer | No | Create from project template |

### Example Request

```bash
curl -X POST "https://public-api.taskip.net/api/public-v1/project" \
  -H "X-Secret-Key: your-secret-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "E-commerce Platform - GreenTech",
    "description": "Build a complete e-commerce platform with inventory management and analytics",
    "client_id": 28,
    "project_manager_id": 3,
    "start_date": "2024-03-01",
    "end_date": "2024-09-01",
    "budget": 120000.00,
    "priority": "high",
    "status": "planning",
    "team_member_ids": [3, 8, 12, 15, 20],
    "tags": ["ecommerce", "platform", "analytics"],
    "custom_fields": {
      "technology_stack": "React, Node.js, MongoDB",
      "deployment_environment": "AWS"
    }
  }'
```

### Python Example
```python
import requests

headers = {
    'X-Secret-Key': 'your-secret-key-here',
    'Content-Type': 'application/json'
}

project_data = {
    "name": "E-commerce Platform - GreenTech",
    "description": "Build a complete e-commerce platform with inventory management and analytics",
    "client_id": 28,
    "project_manager_id": 3,
    "start_date": "2024-03-01",
    "end_date": "2024-09-01",
    "budget": 120000.00,
    "priority": "high",
    "status": "planning",

    "custom_fields": {
        "technology_stack": "React, Node.js, MongoDB",

    'https://public-api.taskip.net/api/public-v1/project',

```

## Create a New Task

Create a new task within a project.

```http
POST /api/public-v1/project/{project_id}/task
```

### Path Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `project_id` | integer | Yes | Project ID |

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | Yes | Task title |
| `description` | string | No | Task description |
| `assigned_to` | integer | No | Team member ID to assign task |

| `tags` | array | No | Array of task tags |

## Log Time Spent on Task

Log time spent working on a specific task.

```http
POST /api/public-v1/task/{id}/time-log
```

### Path Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | integer | Yes | Task ID |

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `hours` | decimal | Yes | Hours spent |
| `date` | string | Yes | Date of work (YYYY-MM-DD) |
| `description` | string | No | Description of work done |
| `billable` | boolean | No | Whether time is billable (default: true) |
| `start_time` | string | No | Start time (HH:MM:SS) |
| `end_time` | string | No | End time (HH:MM:SS) |

### Example Request

```bash
curl -X POST "https://public-api.taskip.net/api/public-v1/task/245/time-log" \
  -H "X-Secret-Key: your-secret-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "hours": 4.5,
    "date": "2024-02-22",
    "description": "Worked on user authentication and product catalog schemas",
    "billable": true,
    "start_time": "09:00:00",
    "end_time": "13:30:00"
  }'
```

## Bulk Task Operations

Perform bulk operations on multiple tasks.

```http
POST /api/public-v1/task/bulk-actions
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `task_ids` | array | Yes | Array of task IDs |
| `action` | string | Yes | Action to perform (`status_update`, `assign`, `delete`, `add_tags`, `set_priority`) |
| `parameters` | object | No | Additional parameters for the action |

#### Action-specific Parameters

**For `status_update` action:**
- `status`: New status for all tasks
- `completion_notes`: Notes for status change

**For `assign` action:**
- `assigned_to`: User ID to assign tasks to

**For `set_priority` action:**
- `priority`: Priority level to set

### Example Request

```bash
curl -X POST "https://public-api.taskip.net/api/public-v1/task/bulk-actions" \
  -H "X-Secret-Key: your-secret-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "task_ids": [240, 241, 242, 243],
    "action": "set_priority",
    "parameters": {
      "priority": "high"
    }
  }'
```

### Common Error Codes

| Status Code | Error Type | Description |
|-------------|------------|-------------|

## Best Practices

### Project Management
1. **Clear objectives**: Define clear project goals and deliverables
2. **Realistic timelines**: Set achievable deadlines with buffer time
3. **Regular updates**: Update project status and progress regularly
4. **Team communication**: Use comments and notifications for team collaboration

### Task Management
1. **Task breakdown**: Break large tasks into smaller, manageable subtasks
2. **Clear assignments**: Assign tasks to specific team members with clear expectations
3. **Priority setting**: Use priority levels to help team members focus on important work
4. **Dependencies tracking**: Track task dependencies to avoid bottlenecks

### Time Tracking
1. **Regular logging**: Log time daily while work is fresh in memory
2. **Detailed descriptions**: Include meaningful descriptions for time entries
3. **Accurate estimates**: Improve estimation accuracy through historical data
4. **Billable vs non-billable**: Properly categorize billable and non-billable time

### Reporting & Analytics
1. **Regular monitoring**: Review project analytics regularly to identify issues early
2. **Team performance**: Use performance data to identify training needs
3. **Budget tracking**: Monitor budget usage to prevent cost overruns
4. **Risk assessment**: Pay attention to risk indicators and take proactive measures
